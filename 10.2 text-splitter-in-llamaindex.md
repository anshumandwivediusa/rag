# Chunking Strategies in LlamaIndex

| **Strategy** | **Best LlamaIndex Tool** | **Reason / Alignment** |
| --- | --- | --- |
| **[Semantic Chunking](ca://s?q=Semantic_chunking_in_LlamaIndex)** | ``SentenceSplitter`` or ``SemanticSplitterNodeParser`` | Splits along logical boundaries (sentences/paragraphs), preserving meaning. |
| **[Recursive Chunking](ca://s?q=Recursive_chunking_in_LlamaIndex)** | ``SentenceSplitter`` with configurable ``chunk_size`` and ``chunk_overlap`` | Allows multiple granularities (sentence, paragraph, section). |
| **[Hierarchical Chunking](ca://s?q=Hierarchical_chunking_in_LlamaIndex)** | Combine ``SentenceSplitter`` + metadata hierarchy | Build summaries at higher levels, chunks at lower levels. |
| **[Entity-Aware Chunking](ca://s?q=Entity_aware_chunking_in_LlamaIndex)** | Custom preprocessing (spaCy/NER) + ``SentenceSplitter`` | Detect entities first, then split around them. |
| **[Graph-Based Chunking](ca://s?q=Graph_based_chunking_in_LlamaIndex)** | ``KnowledgeGraphIndex`` + ``SentenceSplitter`` | Construct graph nodes/edges, then chunk text accordingly. |
| **[Query-Aware Chunking](ca://s?q=Query_aware_chunking_in_LlamaIndex)** | ``RetrieverQueryEngine`` with dynamic spans | Chunks generated at query time, merged dynamically. |
| **[Sliding Window Chunking](ca://s?q=Sliding_window_chunking_in_LlamaIndex)** | ``SentenceSplitter`` with ``chunk_overlap`` | Implements overlapping chunks to preserve boundary context. |
| **[Structure-Preserving Chunking](ca://s?q=Structure_preserving_chunking_in_LlamaIndex)** | ``MarkdownNodeParser``, ``HTMLNodeParser`` | Splits by headings, lists, tags, or code blocks. |
| **[Hybrid Chunking](ca://s?q=Hybrid_chunking_in_LlamaIndex)** | Combination of splitters | Mix recursive + overlap + structure-preserving depending on corpus type. |

## Example

```
from llama_index.core import SimpleDirectoryReader
from llama_index.core.node_parser import SentenceSplitter
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.vector_stores.pinecone import PineconeVectorStore
from llama_index.core import VectorStoreIndex

# 1. Load documents
docs = SimpleDirectoryReader("data/").load_data()

# 2. Split into chunks (semantic/recursive)
splitter = SentenceSplitter(chunk_size=512, chunk_overlap=50)
nodes = splitter.get_nodes_from_documents(docs)

# 3. Embed nodes
embed_model = OpenAIEmbedding()

# 4. Store in vector DB (e.g., Pinecone)
vector_store = PineconeVectorStore(pinecone_index="my-index")
index = VectorStoreIndex(nodes, embed_model=embed_model, vector_store=vector_store)

# 5. Query
query_engine = index.as_query_engine()
response = query_engine.query("What is in this document?")
print(response)
```
