# Document Loader — LangChain RAG Component

The **Document Loader** is the first stage in the **Retrieval-Augmented Generation (RAG)** pipeline.  
It ingests raw data from multiple sources — such as PDFs, text files, web pages, or APIs — and converts them into a standardized format that downstream components (like **Text Splitter** and **Embeddings Model**) can process.

This module ensures that all documents are accessible, clean, and ready for semantic chunking and embedding.

## Core Responsibilities
- **Data Ingestion** → Load documents from local or remote sources.  
- **Format Normalization** → Convert heterogeneous data types (HTML, CSV, JSON, TXT, PDF) into unified text format.  
- **Metadata Extraction** → Capture source info (title, URL, author, date) for traceability.  
- **Preprocessing** → Clean text (remove HTML tags, normalize whitespace, handle encoding).  
- **Output Structuring** → Return a list of `Document` objects, each containing `page_content` and `metadata`.

## Data Structure Categories
LangChain’s Document Loader can process data across three structural levels:

| **Structure Type** | **Description** | **Examples** | **Typical Loader** |
| --- | --- | --- | --- |
| **[Structured Data](ca://s?q=Structured_data_in_LangChain)** | Highly organized, stored in fixed fields or tables. Easy to query and parse. | Databases, Excel, CSV, JSON, XML, YAML | ``SQLDatabaseLoader``, ``UnstructuredExcelLoader``, ``CSVLoader``, ``JSONLoader``, ``UnstructuredXMLLoader``, ``UnstructuredFileLoader`` |
| **[Semi‑Structured Data](ca://s?q=Semi_structured_data_in_LangChain)** | Contains tags or markers but lacks strict schema. | HTML pages, emails, logs, Markdown | ``UnstructuredURLLoader``, ``UnstructuredEmailLoader``, ``MarkdownLoader`` |
| **[Unstructured Data](ca://s?q=Unstructured_data_in_LangChain)** | Free‑form text without predefined structure. | PDFs, DOCX, TXT, scanned images | ``PyPDFLoader``, ``Docx2txtLoader``, ``TextLoader``, ``UnstructuredFileLoader`` |

_Note: Both audio and video must be converted into text before entering the pipeline. Once transcribed: They behave like unstructured text documents. The Document Loader ingests them just like PDFs or DOCX files. Metadata (timestamps, speaker info, video IDs) can be stored in the metadata field for retrieval context._

## Supported Loaders
LangChain provides a variety of built-in loaders under `langchain.document_loaders`.  
| Loader Type | Description | Example |
|--------------|--------------|----------|
| **TextLoader** | Loads plain text files. | `.txt`, `.md` |
| **PyPDFLoader** | Extracts text from PDF pages. | `.pdf` |
| **UnstructuredURLLoader** | Fetches and parses web pages. | `https://example.com` |
| **CSVLoader** | Reads CSV files row by row. | `.csv` |
| **JSONLoader** | Loads structured JSON data. | `.json` |
| **Docx2txtLoader** | Extracts text from Word documents. | `.docx` |

## 🧩 Example Usage

```python
from langchain.document_loaders import PyPDFLoader, TextLoader, UnstructuredURLLoader

# Load from PDF
pdf_loader = PyPDFLoader("data/report.pdf")
pdf_docs = pdf_loader.load()

# Load from text file
txt_loader = TextLoader("data/notes.txt")
txt_docs = txt_loader.load()

# Load from web page
url_loader = UnstructuredURLLoader(["https://example.com/article"])
web_docs = url_loader.load()

# Combine all documents
all_docs = pdf_docs + txt_docs + web_docs
```

Each loader returns a list of `Document` objects:
```python
{
  "page_content": "Extracted text content...",
  "metadata": {"source": "data/report.pdf", "page": 1}
}
```

## 🔗 Integration in RAG Pipeline
The **Document Loader** feeds directly into the **Text Splitter**.  
Typical flow:
```
Document Loader → Text Splitter → Embeddings Model → Vector Store → Retriever → LLM
```

This ensures that every document is properly chunked, embedded, and retrievable for contextual generation.

## 🧰 Best Practices
- Validate file paths and URLs before loading.  
- Use appropriate loader for each data type.  
- Extract metadata for provenance tracking.  
- Handle encoding errors gracefully.  
- Cache loaded documents for large-scale ingestion.  

---

## 🚀 Advanced Tips
- Combine multiple loaders using `RecursiveLoader` for hybrid datasets.  
- Implement custom loaders for proprietary formats (e.g., internal APIs or databases).  
- Integrate with **LangChain Hub** for reusable loader configurations.  
- Use async loaders for large-scale ingestion to improve performance.

## 🧩 Example Custom Loader

```python
from langchain.document_loaders.base import BaseLoader
from langchain.schema import Document

class MyAPILoader(BaseLoader):
    def __init__(self, api_client):
        self.api_client = api_client

    def load(self):
        data = self.api_client.fetch_data()
        return [Document(page_content=item["text"], metadata={"id": item["id"]}) for item in data]
```
