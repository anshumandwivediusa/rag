# 🔍 Postgres Search Cheat Sheet

This guide explains how to use **BM25 keyword search**, **pgvector similarity operators**, and **HNSW indexing** in PostgreSQL.


## 1. Table Definition

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE techcare_docs (
  id bigserial PRIMARY KEY,
  document_text text,
  metadata jsonb,
  embedding_vector vector(1536)  -- dimension depends on your embedding model
);
```


## 2. Insert Example Rows

```sql
INSERT INTO techcare_docs (document_text, metadata, embedding_vector)
VALUES
( 'To factory reset the TechCare X15 laptop...', '{"category":"reset"}', '[0.10, -0.50, 0.75, 0.00]' ),
( 'If your battery drains quickly...', '{"category":"battery"}', '[0.40, 0.10, -0.20, 0.05]' ),
( 'The TechCare X15 supports up to 32GB RAM...', '{"category":"hardware"}', '[0.05, -0.70, 0.20, 0.10]' );
```


## 3. BM25 Keyword Search

BM25 is part of Postgres full‑text search. It matches **exact words**.

```sql
-- Create text search index
CREATE INDEX techcare_docs_text_idx
ON techcare_docs
USING gin(to_tsvector('english', document_text));

-- Query with BM25 ranking
SELECT id, document_text,
       ts_rank_cd(to_tsvector('english', document_text), to_tsquery('reset & laptop')) AS rank
FROM techcare_docs
WHERE to_tsvector('english', document_text) @@ to_tsquery('reset & laptop')
ORDER BY rank DESC
LIMIT 3;
```


## 4. Embedding Search with pgvector Operators

```sql
-- Euclidean distance
SELECT id, document_text
FROM techcare_docs
ORDER BY embedding_vector <-> '[0.11, -0.52, 0.77, 0.01]'
LIMIT 3;

-- Cosine similarity (best for semantic embeddings)
SELECT id, document_text
FROM techcare_docs
ORDER BY embedding_vector <#> '[0.11, -0.52, 0.77, 0.01]'
LIMIT 3;

-- Inner product
SELECT id, document_text
FROM techcare_docs
ORDER BY embedding_vector <=> '[0.11, -0.52, 0.77, 0.01]'
LIMIT 3;
```


## 5. HNSW Index for Fast ANN Search

```sql
-- Create HNSW index for cosine similarity
CREATE INDEX techcare_docs_embedding_hnsw
ON techcare_docs
USING hnsw (embedding_vector vector_cosine_ops)
WITH (m = 16, ef_construction = 64);

-- Tune search recall
SET hnsw.ef_search = 100;

-- Query with HNSW acceleration
SELECT id, document_text
FROM techcare_docs
ORDER BY embedding_vector <#> '[0.11, -0.52, 0.77, 0.01]'
LIMIT 3;
```


## 6. Summary Table

| **Method** | **Type** | **Usage in Postgres** | **Best For** |
|------------|----------|-----------------------|--------------|
| **BM25** | Keyword search | `ts_rank_cd(...) ORDER BY rank` | Exact word matches |
| **<->** | Euclidean distance | `ORDER BY embedding_vector <-> '[...]'` | Geometric closeness |
| **<#>** | Cosine similarity | `ORDER BY embedding_vector <#> '[...]'` | Semantic embeddings |
| **<=>** | Inner product | `ORDER BY embedding_vector <=> '[...]'` | Recommendations |
| **HNSW** | ANN index | `CREATE INDEX ... USING hnsw` | Fast search at scale |


## ✅ Takeaway
- **BM25** → keyword search (exact words).  
- **Cosine `<#>`** → semantic search (meaning).  
- **HNSW** → makes embedding search fast at scale.  
- Operators = *how similarity is measured*.  
- Indexes (BM25, HNSW) = *how results are retrieved efficiently*.

# ChromaDB

## 1. Install ChromaDB

```bash
pip install chromadb
```



## 2. Basic Setup

```python
import chromadb
from chromadb.config import Settings

# Create a client
client = chromadb.Client(Settings(
    persist_directory="./chroma_store"  # optional: persist to disk
))

# Create a collection
collection = client.create_collection(name="techcare_docs")
```



## 3. Insert Example Documents

```python
collection.add(
    documents=[
        "To factory reset the TechCare X15 laptop...",
        "If your battery drains quickly...",
        "The TechCare X15 supports up to 32GB RAM..."
    ],
    metadatas=[
        {"category": "reset"},
        {"category": "battery"},
        {"category": "hardware"}
    ],
    ids=["1", "2", "3"],
    embeddings=[
        [0.10, -0.50, 0.75, 0.00],
        [0.40, 0.10, -0.20, 0.05],
        [0.05, -0.70, 0.20, 0.10]
    ]
)
```



## 4. Query by Cosine Similarity

```python
results = collection.query(
    query_embeddings=[[0.11, -0.52, 0.77, 0.01]],
    n_results=3
)

print(results)
```

👉 Returns the **top‑3 nearest neighbors** by cosine similarity.  
Document 1 (“factory reset”) will rank first because it’s closest in meaning.



## 5. Hybrid Search (Text + Embeddings)

ChromaDB also supports querying by **text** (it will embed internally):

```python
results = collection.query(
    query_texts=["reset laptop"],
    n_results=3
)

print(results)
```

👉 Converts `"reset laptop"` into an embedding and finds nearest neighbors.



## 6. Summary Table

| **Method** | **ChromaDB Usage** | **Best For** |
|------------|---------------------|--------------|
| **Cosine similarity** | `collection.query(query_embeddings=...)` | Semantic search |
| **Text query** | `collection.query(query_texts=...)` | Easy queries without manual embeddings |
| **Metadata filters** | `collection.query(where={"category":"reset"})` | Hybrid search (semantic + structured) |
| **Persistence** | `persist_directory="./chroma_store"` | Save collections to disk |



## ✅ Takeaway
- **ChromaDB** is embedding‑native (semantic search by default).  
- You query with either **embeddings** or **raw text**.  
- Cosine similarity is the default metric.  
- Metadata filters let you combine semantic search with structured queries.  
- Unlike Postgres, you don’t need BM25 or HNSW setup — ChromaDB handles vector search internally.
