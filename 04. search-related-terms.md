# **Search‑Related Concepts With Real Examples**

## **1. BM25 — Keyword‑Based Ranking Using TF‑IDF + Length Normalization**

### **Concept**  
BM25 scores documents based on:  
- How often query words appear (TF)  
- How rare those words are (IDF)  
- How long the document is (length normalization)

### **Example**  
Query: **“factory reset laptop”**

Documents:  
1. “How to factory reset a laptop using F11”  
2. “Laptop battery draining too fast”  
3. “Factory settings for TechCare devices”

BM25 ranks them:  
1 → highest (contains all 3 words)  
3 → medium (contains “factory”)  
2 → lowest (only “laptop”)

## **2. TF‑IDF — Term Frequency × Inverse Document Frequency**

### **Concept**  
- **TF** → how often a word appears in a document  
- **IDF** → how rare the word is across all documents  
- Rare + frequent = important

### **Example**  
Query: **“boot error”**

Document A: “Boot error occurs when BIOS fails.”  
Document B: “Laptop battery error message.”

TF‑IDF gives higher weight to:  
- “boot” (rare word)  
- “error” (common word → lower weight)

Document A scores higher because “boot” is rare and relevant.

## **3. Inverted Index — Word → List of Documents**

### **Concept**  
A dictionary where each word points to all documents containing it.

### **Example**  
Documents:  
1. “reset laptop factory”  
2. “battery laptop drain”  
3. “factory settings restore”

Inverted index:

| Word | Docs |
|------|------|
| reset | 1 |
| laptop | 1, 2 |
| factory | 1, 3 |
| battery | 2 |

Query: **“factory laptop”**  
→ intersection of docs for “factory” and “laptop”  
→ Document **1**



## 4. The Relationship: TF + IDF → TF‑IDF → BM25
Think of it like this:

  - TF = how often a word appears
  
  - IDF = how rare/important the word is
  
  - TF‑IDF = TF × IDF
  
  - BM25 = improved TF‑IDF + document length normalization + saturation

BM25 is basically a smarter, more realistic version of TF‑IDF.

## **4. Boolean Search — AND / OR / NOT**

### **Concept**  
Uses logical operators to include or exclude terms.

### **Examples**

- **AND**  
  Query: `factory AND reset`  
  → must contain both words  
  → Document 1

- **OR**  
  Query: `factory OR battery`  
  → documents containing either  
  → Documents 1, 2, 3

- **NOT**  
  Query: `laptop NOT battery`  
  → documents with “laptop” but not “battery”  
  → Document 1

Boolean search is strict — no semantic understanding.

---

## **5. Lexical Search — Pure Keyword Matching**

### **Concept**  
Searches based on **exact words**, not meaning.  
BM25, TF‑IDF, Boolean search all fall under lexical search.

### **Example**  
Query: **“won’t turn on”**

Document: “Laptop does not power on.”

Lexical search fails because:  
- “won’t” ≠ “does not”  
- “turn on” ≠ “power on”

Semantic search would succeed, but lexical search would not.

---

# ⭐ **One‑Sentence Summary**  
> BM25, TF‑IDF, inverted index, Boolean search, and lexical search are all **keyword‑based retrieval methods** that match exact words rather than meaning.
